<task type="auto">
  <name>Task 2: Extend SheetsClient to support alternate spreadsheet and add tests</name>
  <files>src/negotiation/sheets/client.py, tests/sheets/test_client.py</files>
  <action>
1. In `src/negotiation/sheets/client.py`, modify the three public methods (`get_all_influencers`, `find_influencer`, `get_pay_range`) to accept an optional `spreadsheet_key_override: str | None = None` parameter.

2. Add a private helper method `_get_spreadsheet_for(self, spreadsheet_key_override: str | None = None) -> gspread.Spreadsheet` that:
   - If `spreadsheet_key_override` is None or empty, calls the existing `_get_spreadsheet()` (returns cached master sheet)
   - If `spreadsheet_key_override` is provided, calls `self._gc.open_by_key(spreadsheet_key_override)` directly (no caching -- different campaigns may use different sheets)

3. Update `get_all_influencers` to call `self._get_spreadsheet_for(spreadsheet_key_override)` instead of `self._get_spreadsheet()`. Same for the chain through `find_influencer` and `get_pay_range` -- pass spreadsheet_key_override through.

4. In `tests/sheets/test_client.py`, add a new test class `TestSpreadsheetKeyOverride` with tests:
   - `test_override_opens_different_spreadsheet`: When spreadsheet_key_override is provided, `gc.open_by_key` is called with the override key (not the default key).
   - `test_no_override_uses_default`: When spreadsheet_key_override is None, the default spreadsheet is used (existing behavior).
   - `test_override_does_not_cache`: Two calls with different overrides result in two separate `open_by_key` calls.

All existing tests must continue to pass unchanged since the new parameter defaults to None.
  </action>
  <verify>Run `cd src && python -c "from negotiation.sheets.client import SheetsClient; import inspect; sig=inspect.signature(SheetsClient.find_influencer); assert 'spreadsheet_key_override' in sig.parameters"` to confirm signature. Run `python -m pytest tests/sheets/test_client.py -v` to confirm all tests pass.</verify>
  <done>SheetsClient.find_influencer, get_all_influencers, and get_pay_range all accept spreadsheet_key_override parameter. Default behavior (None) unchanged -- uses master spreadsheet. When override provided, opens the specified spreadsheet. New tests verify override behavior.</done>
</task>
