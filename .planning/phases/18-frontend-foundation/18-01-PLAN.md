<task type="auto">
  <name>Task 1: Initialize React + Vite + TypeScript project</name>
  <files>
    frontend/package.json
    frontend/tsconfig.json
    frontend/tsconfig.app.json
    frontend/tsconfig.node.json
    frontend/vite.config.ts
    frontend/index.html
    frontend/src/main.tsx
    frontend/src/App.tsx
    frontend/src/vite-env.d.ts
    frontend/.gitignore
  </files>
  <action>
    Create the frontend/ directory at the project root. Use `npm create vite@latest frontend -- --template react-ts` to scaffold the project. If the interactive prompt is problematic, manually create the files instead:

    1. `frontend/package.json` with dependencies:
       - react ^19, react-dom ^19 (dependencies)
       - @vitejs/plugin-react, vite, typescript, @types/react, @types/react-dom (devDependencies)
       - Scripts: dev, build, preview, lint

    2. `frontend/vite.config.ts`:
       - Import react plugin from @vitejs/plugin-react
       - Export defineConfig with plugins: [react()]
       - Add server.proxy config: proxy `/api` requests to `http://localhost:8000` (for dev mode). This allows the Vite dev server to forward API calls to the FastAPI backend.

    3. `frontend/index.html`:
       - Standard Vite HTML with `<div id="root"></div>` and `<script type="module" src="/src/main.tsx"></script>`

    4. `frontend/src/main.tsx`:
       - Import React, ReactDOM, App, and index.css
       - Render `<App />` into `document.getElementById('root')` using createRoot

    5. `frontend/src/App.tsx`:
       - Simple functional component that renders a dashboard placeholder:
         - An h1 "Campaign Dashboard"
         - A paragraph "Influencer negotiation management"
         - Use Tailwind classes for styling: centered layout, text sizing, padding

    6. `frontend/src/vite-env.d.ts`:
       - Standard Vite client types reference: `/// <reference types="vite/client" />`

    7. `frontend/tsconfig.json`, `frontend/tsconfig.app.json`, `frontend/tsconfig.node.json`:
       - Standard Vite React-TS tsconfig setup with strict mode, JSX react-jsx

    8. `frontend/.gitignore`:
       - node_modules/, dist/, *.local

    Run `cd frontend && npm install` to install all dependencies.
  </action>
  <verify>
    Run `cd frontend && npm run build` and confirm it exits 0 and produces `frontend/dist/index.html`.
  </verify>
  <done>frontend/ directory exists with all scaffold files, npm install succeeds, npm run build produces dist/ with index.html and bundled JS/CSS assets.</done>
</task>

<task type="auto">
  <name>Task 2: Add Tailwind CSS with PostCSS configuration</name>
  <files>
    frontend/tailwind.config.js
    frontend/postcss.config.js
    frontend/src/index.css
  </files>
  <action>
    1. Install Tailwind CSS and its dependencies:
       `cd frontend && npm install -D tailwindcss @tailwindcss/postcss postcss`

    2. Create `frontend/postcss.config.js`:
       ```js
       export default {
         plugins: {
           "@tailwindcss/postcss": {},
         },
       };
       ```

    3. Create `frontend/tailwind.config.js`:
       ```js
       /** @type {import('tailwindcss').Config} */
       export default {
         content: [
           "./index.html",
           "./src/**/*.{js,ts,jsx,tsx}",
         ],
         theme: {
           extend: {},
         },
         plugins: [],
       };
       ```

    4. Create `frontend/src/index.css` with Tailwind directives:
       ```css
       @import "tailwindcss";
       ```

    5. Verify Tailwind is working by updating `frontend/src/App.tsx` to use Tailwind utility classes if not already done in Task 1:
       - Container with `min-h-screen bg-gray-50 flex flex-col items-center justify-center`
       - h1 with `text-3xl font-bold text-gray-900`
       - p with `mt-2 text-gray-600`

    6. Run `cd frontend && npm run build` to confirm Tailwind processes correctly (the built CSS in dist/ should contain the utility classes used, not the raw directives).
  </action>
  <verify>
    Run `cd frontend && npm run build` and confirm exit 0. Check that `frontend/dist/assets/*.css` contains actual CSS rules (not just @tailwind directives). Run `grep -l "bg-gray" frontend/dist/assets/*.css` to confirm Tailwind compiled the utilities.
  </verify>
  <done>Tailwind CSS is configured and integrated. Build output contains compiled Tailwind utilities. App.tsx uses Tailwind classes for styling.</done>
</task>
