Implementation:

```python
async def run_sheet_monitor_loop(services: dict[str, Any]) -> None:
```

1. **Setup**: Extract from `services`: `sheets_client`, `negotiation_states`, `slack_notifier`, `state_conn` (the SQLite connection). Create a `SheetMonitor(sheets_client, state_conn)`. Call `init_processed_influencers_table(state_conn)` to ensure table exists. Log "Sheet monitor started".

2. **Infinite loop** with `await asyncio.sleep(3600)` at the end of each iteration (poll hourly):
   a. **Extract active campaigns**: Iterate `negotiation_states.values()`, collect unique `Campaign` objects by `campaign_id` (deduplicate since multiple influencers share a campaign). Only include campaigns that have `influencer_sheet_tab` or `influencer_sheet_id` set (campaigns without sheet routing have no tab to monitor).
   b. **For each unique campaign**: Call `monitor.check_campaign_sheet(campaign)` to get a `SheetDiff`.
   c. **Handle new rows (MON-02)**: If `diff.new_rows` is not empty:
      - Convert new InfluencerRows to the `found_influencers` format expected by `start_negotiations_for_campaign`: `[{"name": row.name, "sheet_data": row}]`.
      - Import and call `start_negotiations_for_campaign(found_influencers, campaign, services)` (import from `negotiation.app` -- use a late import inside the function to avoid circular imports).
      - After successful negotiation start, call `monitor.mark_rows_processed(campaign.campaign_id, diff.new_rows)` to record them as processed.
      - Log: "Sheet monitor: {N} new influencers found for campaign {campaign_id}".
   d. **Handle modified rows (MON-03)**: If `diff.modified_rows` is not empty:
      - For each (row, old_hash) in `diff.modified_rows`:
        - Post a Slack alert via `slack_notifier.post_escalation()` with blocks showing: campaign name, influencer name, and a note that the row was modified after negotiation started. Use fallback_text like: "Influencer row modified: {name} in campaign {campaign.client_name}".
        - Update the stored hash: call `monitor._mark_processed(campaign.campaign_id, row.name, monitor._compute_row_hash(row))`.
      - Log: "Sheet monitor: {N} modified rows in campaign {campaign_id}".
   e. **Also handle initial ingestion marking**: At the START of the first poll (or whenever a campaign is first seen by the monitor), the existing influencers that were already negotiated should be in the processed table. To handle this, before checking for new rows, look at `negotiation_states` to find influencer names that are already in negotiations for this campaign. Mark those as processed if not already tracked. This prevents re-sending outreach for influencers ingested during initial campaign creation (before the monitor existed). Implementation: iterate `negotiation_states`, for each entry where `entry["campaign"].campaign_id == campaign.campaign_id`, get the influencer name from `entry["context"]["influencer_name"]`, and call `_mark_processed` with a placeholder hash if not already in the processed table.

3. **Error handling**: Wrap the entire per-campaign check in try/except. Log exceptions as warnings but continue to the next campaign. Never let one campaign's error crash the loop.

4. **Imports needed at top of file**: `asyncio`, `structlog`, `from typing import Any`. Late-import `start_negotiations_for_campaign` inside the function body to avoid circular imports.
