  <files>src/negotiation/api/negotiations.py</files>
  <action>
Create a new FastAPI router module `src/negotiation/api/negotiations.py` with two endpoints:

**Endpoint 1: GET /campaigns/{campaign_id}/negotiations**
- Filters `request.app.state.negotiation_states` by entries whose `context["campaign_id"]` matches the path param
- For each matching thread entry, extract:
  - `thread_id`: the dict key from negotiation_states
  - `influencer_name`: from `entry["context"]["influencer_name"]`
  - `influencer_email`: from `entry["context"].get("influencer_email", "")`
  - `state`: `str(entry["state_machine"].state)` (the NegotiationState value)
  - `round_count`: `entry["round_count"]`
  - `counterparty_type`: `entry["context"].get("counterparty_type", "direct_influencer")`
  - `agency_name`: `entry["context"].get("agency_name")` (nullable)
  - `current_rate`: Extract from cpm_tracker if available. Use `entry.get("cpm_tracker")` — if it has a `running_average_cpm` attribute and it is not None, use `float(running_average_cpm)`, else None.
- Return Pydantic response model `CampaignDetailResponse` with fields:
  - `campaign_id: str`
  - `negotiations: list[NegotiationSummary]` (the per-influencer items)
  - `total: int`
- If no negotiations match, return empty list with total=0 (not a 404)

**Endpoint 2: GET /campaigns/{campaign_id}/negotiations/{thread_id}/timeline**
- First verify the thread_id exists in negotiation_states and belongs to the campaign_id (return 404 if not)
- Get state machine history from `entry["state_machine"].history` — this returns `list[tuple[NegotiationState, str, NegotiationState]]` (from_state, event, to_state)
- Convert to list of `StateTransition` objects with fields: `from_state: str`, `event: str`, `to_state: str`
- Query audit trail using `query_audit_trail` from `negotiation.audit.store`:
  - Get the audit DB connection from `request.app.state` — it's stored as `services["audit_conn"]` during startup. Access it via `getattr(request.app.state, "services", {}).get("audit_conn")`
  - Call `query_audit_trail(conn, campaign_id=campaign_id, influencer_name=influencer_name, limit=200)` (use influencer_name from the thread's context since thread_id is not directly queryable in the audit schema... actually the audit_log DOES have a thread_id column, so filter by thread_id instead)
  - Actually: the `query_audit_trail` function only supports filtering by `influencer_name`, `campaign_id`, `from_date`, `to_date`, `event_type`. It does NOT support thread_id filtering. So query by `campaign_id` AND `influencer_name` to get the right entries.
  - Convert results to `TimelineEntry` objects: `timestamp: str`, `event_type: str`, `direction: str | None`, `email_body: str | None`, `negotiation_state: str | None`, `rates_used: str | None`, `metadata: dict | None`
- Return `TimelineResponse` with fields:
  - `thread_id: str`
  - `influencer_name: str`
  - `state_transitions: list[StateTransition]`
  - `timeline: list[TimelineEntry]`

Pydantic models to define in the file (before the endpoints):
- `NegotiationSummary(BaseModel)` — per-influencer row
- `CampaignDetailResponse(BaseModel)` — wraps list
- `StateTransition(BaseModel)` — from state machine history
- `TimelineEntry(BaseModel)` — from audit trail
- `TimelineResponse(BaseModel)` — wraps transitions + timeline

Use `from __future__ import annotations` at top. Import patterns match campaigns.py (APIRouter, Request, BaseModel).
