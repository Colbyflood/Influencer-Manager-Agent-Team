**In `frontend/src/types/campaign.ts`**, add these new types (keep all existing types):

```typescript
export interface NegotiationSummary {
  thread_id: string;
  influencer_name: string;
  influencer_email: string;
  state: string;
  round_count: number;
  counterparty_type: string;
  agency_name: string | null;
  current_rate: number | null;
}

export interface CampaignDetailResponse {
  campaign_id: string;
  negotiations: NegotiationSummary[];
  total: number;
}

export interface StateTransition {
  from_state: string;
  event: string;
  to_state: string;
}

export interface TimelineEntry {
  timestamp: string;
  event_type: string;
  direction: string | null;
  email_body: string | null;
  negotiation_state: string | null;
  rates_used: string | null;
  metadata: Record<string, string> | null;
}

export interface TimelineResponse {
  thread_id: string;
  influencer_name: string;
  state_transitions: StateTransition[];
  timeline: TimelineEntry[];
}
```

**Create `frontend/src/components/CampaignDetail.tsx`**:

A component that receives `campaignId: string` and `onBack: () => void` as props.

- On mount, fetch `/api/v1/campaigns/${campaignId}/negotiations` and store the `CampaignDetailResponse`
- Show a back button/link at top that calls `onBack`
- Show campaign ID as a heading
- Render a table of influencer negotiations with columns: Influencer Name, State, Rate, Rounds, Counterparty Type, Agency
- State column: Use colored badges (green for agreed, blue for active states like awaiting_reply/counter_sent/counter_received/initial_offer, amber for escalated, red for rejected, gray for stale)
- Rate column: Show as `$X.XX` or `--` if null
- Each row is clickable — clicking sets `selectedThreadId` state, which triggers rendering the NegotiationTimeline component (created in Task 2)
- Loading state: spinner (same pattern as CampaignList)
- Error state: red error box (same pattern as CampaignList)
- Empty state: "No negotiations found for this campaign"

Use Tailwind classes consistent with the existing CampaignCard/CampaignList styling (bg-white, rounded-lg, shadow-sm, border-gray-200, etc.).

**Create `frontend/src/components/NegotiationTimeline.tsx`**:

A component that receives `campaignId: string`, `threadId: string`, `onBack: () => void` as props.

- On mount, fetch `/api/v1/campaigns/${campaignId}/negotiations/${threadId}/timeline` and store the `TimelineResponse`
- Show a back button at top that calls `onBack`
- Show influencer name as heading
- **State Transitions section**: Render the `state_transitions` array as a vertical visual flow. For each transition show: `from_state` -> (event) -> `to_state`. Use arrow indicators or a vertical timeline layout. Color-code states same as the detail table badges.
- **Activity Timeline section**: Render the `timeline` array (sorted by timestamp, newest first as returned by API) as a chronological list. For each entry show:
  - Timestamp (formatted as relative or short date)
  - Event type badge (email_sent=blue, email_received=green, state_transition=gray, escalation=amber, agreement=green-bold)
  - Direction indicator if email (sent/received arrow)
  - Email body in a collapsible/expandable section (truncated to first 200 chars by default, click to expand) — skip if null
  - Rates used if present
  - For state_transition entries: show from_state -> to_state from metadata
- Loading/error/empty states follow same patterns as other components

**Update `frontend/src/components/CampaignCard.tsx`**:
- Add an `onSelect?: (campaignId: string) => void` prop
- Make the card clickable: add `onClick={() => onSelect?.(campaign.campaign_id)}` to the outer div
- Add cursor-pointer class and hover:shadow-md transition

**Update `frontend/src/App.tsx`**:
- Add state: `selectedCampaignId: string | null` (initially null)
- When `selectedCampaignId` is set, render `<CampaignDetail campaignId={selectedCampaignId} onBack={() => setSelectedCampaignId(null)} />` instead of `<CampaignList />`
- Pass `onSelect={setSelectedCampaignId}` to CampaignList (and CampaignList passes it through to CampaignCard)
- Update CampaignList to accept and forward an `onSelect` prop to each CampaignCard

This gives simple client-side navigation: List -> Detail -> Timeline, with back buttons at each level. No React Router needed — just conditional rendering based on state.
