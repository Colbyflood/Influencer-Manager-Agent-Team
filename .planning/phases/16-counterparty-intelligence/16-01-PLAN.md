  <behavior>
    classify_counterparty(from_email: str, email_body: str, subject: str = "") -> CounterpartyProfile

    Detection signals (checked in order, accumulated):
    1. Domain analysis: known agency domains (unitedtalent.com, wmgagent.com, caa.com, icmpartners.com, paradigmagency.com, etc.) -> talent_manager. Personal domains (gmail.com, yahoo.com, hotmail.com, outlook.com, icloud.com) -> direct_influencer. Custom domains -> neutral (could be either).
    2. Signature keywords: scan last 10 lines of email_body for title indicators. Manager/agent titles ("Manager", "Talent Manager", "Agent", "Talent Agent", "Talent Director", "VP Talent", "Booking", "on behalf of", "representing") -> talent_manager signal. Creator titles ("Creator", "Content Creator", "Influencer") -> direct_influencer signal.
    3. Email structure: formal salutation + sign-off patterns ("Best regards", "Warm regards" with name + title) -> slight talent_manager signal. Casual ("hey!", "thanks!") -> slight direct_influencer signal.

    Confidence scoring:
    - 2+ talent_manager signals -> CounterpartyType.TALENT_MANAGER, confidence 0.9
    - 1 strong signal (agency domain) -> TALENT_MANAGER, confidence 0.8
    - 1 weak signal (signature keyword only) -> TALENT_MANAGER, confidence 0.6
    - No signals or conflicting -> DIRECT_INFLUENCER, confidence 0.5 (default)
    - 1+ direct_influencer signals, no talent_manager -> DIRECT_INFLUENCER, confidence 0.8

    Agency name extraction: if talent_manager detected, attempt to extract agency name from:
    - Email domain (e.g., "unitedtalent.com" -> "United Talent Agency")
    - Signature block text (look for company name patterns)

    Test cases:
    - "sarah@unitedtalent.com" with body "Hi, I'm reaching out on behalf of Jake..." -> TALENT_MANAGER, confidence 0.9, agency="United Talent Agency"
    - "jake.creator@gmail.com" with casual body -> DIRECT_INFLUENCER, confidence 0.8
    - "contact@customdomain.com" with "Best regards, Sarah - Talent Manager" -> TALENT_MANAGER, confidence 0.8
    - "info@customdomain.com" with neutral body -> DIRECT_INFLUENCER, confidence 0.5 (default)
    - "manager@company.com" with "Talent Director" in signature -> TALENT_MANAGER, confidence 0.8
  </behavior>
  <implementation>
    Models (models.py):
    - CounterpartyType(StrEnum): DIRECT_INFLUENCER = "direct_influencer", TALENT_MANAGER = "talent_manager"
    - DetectionSignal(BaseModel, frozen=True): signal_type (str, e.g. "agency_domain", "signature_title", "email_structure"), value (str, the matched text), strength (float, 0.0-1.0), indicates (CounterpartyType)
    - CounterpartyProfile(BaseModel, frozen=True): counterparty_type (CounterpartyType), confidence (float), signals (list[DetectionSignal]), agency_name (str | None), contact_name (str | None, extracted from signature), contact_title (str | None, extracted from signature)

    Classifier (classifier.py):
    - KNOWN_AGENCY_DOMAINS: dict[str, str] mapping domain -> agency name (10-15 entries)
    - PERSONAL_DOMAINS: set of known personal email domains
    - MANAGER_TITLE_PATTERNS: list of regex patterns for title detection
    - classify_counterparty(from_email, email_body, subject="") -> CounterpartyProfile
    - Internal helpers: _check_domain(email) -> DetectionSignal | None, _scan_signature(body) -> list[DetectionSignal], _assess_structure(body) -> DetectionSignal | None, _extract_agency_name(signals) -> str | None
    - Aggregate signals, compute confidence, return CounterpartyProfile
  </implementation>
</feature>
