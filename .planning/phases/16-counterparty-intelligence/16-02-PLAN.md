  <action>
    Create ThreadContactTracker that manages per-thread contact lists:

    ThreadContact(BaseModel, frozen=True):
    - email: str
    - name: str | None (extracted from From header display name)
    - counterparty_type: CounterpartyType
    - title: str | None (from classifier's contact_title)
    - first_seen_at: str (ISO 8601 timestamp)
    - is_primary: bool (True for first contact on thread)

    ThreadContactTracker:
    - __init__(): self._threads: dict[str, ThreadContactRegistry] where ThreadContactRegistry is a simple dataclass with fields: contacts (dict[str, ThreadContact] keyed by lowercase email), agency_name (str | None), primary_counterparty_type (CounterpartyType)
    - update(thread_id: str, from_email: str, profile: CounterpartyProfile) -> ThreadContact: Adds or updates a contact for the thread. If this is the first contact, marks as primary and sets primary_counterparty_type. If a talent_manager is detected on a thread that was previously direct_influencer, upgrades primary_counterparty_type to talent_manager. Stores agency_name if detected. Returns the ThreadContact created/updated.
    - get_contacts(thread_id: str) -> list[ThreadContact]: Returns all contacts for a thread, primary first.
    - get_primary_type(thread_id: str) -> CounterpartyType: Returns the thread's primary counterparty type (talent_manager if any manager detected, else direct_influencer).
    - get_agency_name(thread_id: str) -> str | None: Returns agency name if detected.
    - has_multiple_contacts(thread_id: str) -> bool: True if more than one distinct email address has sent on this thread.

    Tests must cover:
    - Single contact added, is_primary=True
    - Second contact (assistant) added, is_primary=False, original remains primary
    - Manager detected upgrades thread type from direct_influencer to talent_manager
    - Agency name persisted from profile
    - get_contacts returns primary first
    - has_multiple_contacts returns True after second contact
  </action>

  <done>ThreadContactTracker correctly tracks multiple contacts per thread, preserves primary contact, stores agency name, and reports thread counterparty type</done>
