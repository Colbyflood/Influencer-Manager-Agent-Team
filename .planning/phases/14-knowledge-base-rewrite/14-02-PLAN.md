**Update src/negotiation/llm/knowledge_base.py:**

Add a new function `load_examples_for_stage(stage: str, platform: str | None = None, kb_dir: Path = DEFAULT_KB_DIR) -> str` that:

1. Scans `kb_dir / "examples"` directory for all `.md` files
2. For each file, parses YAML frontmatter (between `---` delimiters) to extract `scenario`, `title`, `stages`, `tactics`, and `platform` fields
3. Filters examples where `stage` is in the example's `stages` list
4. Optionally filters by `platform` (if platform is not None, include examples where example platform is None/null OR matches the given platform)
5. Returns the filtered examples concatenated as a markdown string, with each example preceded by its title as a heading
6. If no examples match, returns an empty string (not an error)

Use `yaml.safe_load()` for frontmatter parsing (PyYAML is already a transitive dependency via Pydantic -- verify with `python -c "import yaml"` first). If yaml is not available, parse the simple YAML frontmatter manually with string splitting (the frontmatter is simple key-value and list fields, no complex nesting needed).

Frontmatter parsing approach:
- Split file content on `---` (first occurrence after start)
- Parse the YAML block between the two `---` delimiters
- The rest after the second `---` is the markdown content

**Update the existing `load_knowledge_base` function signature** to accept an optional `stage: str | None = None` parameter. When stage is provided:
- Load general.md + platform.md as before (the playbook)
- Also call `load_examples_for_stage(stage, platform)` and append the result as a new section
- The combined output should have the playbook first, then a `## Relevant Email Examples` heading, then the filtered examples

When stage is None (default), behave exactly as before (backward compatible -- just general + platform content, no examples). This preserves all existing call sites.

**Update src/negotiation/llm/prompts.py:**

No changes needed to the prompt template itself -- the `{knowledge_base_content}` placeholder already receives whatever load_knowledge_base returns. The examples will be part of that content when stage is provided.

However, add a brief instruction to EMAIL_COMPOSITION_SYSTEM_PROMPT after the KNOWLEDGE BASE section, before the RULES section:
```
If email examples are provided below, use them as STYLE REFERENCE only. Match the tone, structure, and
tactics demonstrated -- but never copy verbatim. Adapt to the specific negotiation context.
```

**Do NOT modify composer.py or negotiation_loop.py in this task** -- they currently call `load_knowledge_base(platform)` without stage. The caller updates happen in Task 2.
