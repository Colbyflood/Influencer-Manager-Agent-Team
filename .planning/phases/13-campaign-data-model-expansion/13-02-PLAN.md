<task type="auto">
  <name>Task 1: Expand parse_custom_fields and build_campaign for all 42 fields</name>
  <files>src/negotiation/campaign/ingestion.py</files>
  <action>
Read the existing ingestion.py first. Modify the following functions:

**1. Expand `parse_custom_fields`** to handle new ClickUp field types:
- Add handling for `select` fields: extract the selected option's `name` from ClickUp's `{value: {id, name, ...}}` structure for select/dropdown fields. ClickUp select fields have `value` as an object with `name` key, or sometimes as an integer index -- handle both.
- Add handling for `multi_select` fields: extract list of option names from ClickUp's `{value: [{id, name}, ...]}` array structure.
- Add handling for `yes_no` / boolean fields: convert "Yes"/"No" strings and ClickUp's boolean representations to Python bool.
- Add handling for `date_range` fields: ClickUp date ranges come as `{start: unix_ms, end: unix_ms}` -- convert to ISO date strings.
- Add handling for `duration_select` fields: map ClickUp select values like "30 Days", "60 Days", "90 Days", "6 Months", "1 Year", "Perpetual", "Not required" to `UsageRightsDuration` enum values.
- Load `field_types` section from the YAML config to know which fields need special parsing.

**2. Add `_resolve_dot_path`** helper function:
Given a parsed flat dict like `{"goals.primary_goal": "Organic Social Performance", "budget_constraints.cpm_target": 25}`, resolve dot-separated paths into a nested dict structure: `{"goals": {"primary_goal": "Organic Social Performance"}, "budget_constraints": {"cpm_target": 25}}`. This bridges the flat ClickUp field mapping to the nested Pydantic model structure.

**3. Expand `build_campaign`** to construct all sub-models:
After resolving dot paths, construct each sub-model from its nested dict:
- `CampaignBackground(**nested.get("background", {}))`
- `CampaignGoals(**nested.get("goals", {}))`
- `DeliverableScenarios(target_deliverables=..., **nested.get("deliverables", {}))`
- `UsageRights(target=UsageRightsSet(**...), minimum=UsageRightsSet(**...))`
- `BudgetConstraints(**nested.get("budget_constraints", {}))`
- `ProductLeverage(**nested.get("product_leverage", {}))`
- `CampaignRequirements(**nested.get("requirements", {}))`
- `DistributionInfo(**nested.get("distribution", {}))`

Keep all existing Campaign fields (`budget`, `target_deliverables`, `cpm_range`, `platform`, `timeline`, `influencers`) populated the same way for backward compatibility.

For monetary Decimal fields in sub-models, use the same `Decimal(str(value))` pattern as existing code.

**4. Update `load_field_mapping`** to also return `field_types` from the YAML config (change return type to include field_types dict, or add a separate `load_field_types` function).

Keep backward compatibility: if `field_types` section is missing from YAML, return empty dict.
  </action>
