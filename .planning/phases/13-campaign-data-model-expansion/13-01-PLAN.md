Expand the Campaign Pydantic model from 8 fields to all 42 ClickUp form fields, creating structured sub-models for goals, deliverable scenarios, usage rights, budget constraints, product leverage, and campaign requirements.

Output: Expanded `models.py` with 7+ new Pydantic sub-models, updated `campaign_fields.yaml` mapping all 42 fields, and comprehensive model validation tests.

<task type="auto">
  <name>Task 1: Create sub-models and expand Campaign model for all 42 fields</name>
  <files>src/negotiation/campaign/models.py</files>
  <action>
Add the following new Pydantic models to `models.py`, all with `frozen=True` ConfigDict and Decimal for monetary values (reject floats like existing patterns):

1. **`UsageRightsDuration`** (StrEnum): `not_required`, `days_30`, `days_60`, `days_90`, `months_6`, `year_1`, `perpetual`

2. **`OptimizeFor`** (StrEnum): `cpm`, `content_volume_usage_rights`, `balance`

3. **`CampaignGoals`** (BaseModel, frozen):
   - `primary_goal: str` (Campaign Type / Primary Goal - select)
   - `secondary_goal: str | None = None`
   - `business_context: str | None = None`
   - `optimize_for: OptimizeFor = OptimizeFor.balance`

4. **`DeliverableScenarios`** (BaseModel, frozen):
   - `target_deliverables: list[str]` (multi-select of deliverable types)
   - `content_syndication: bool = False`
   - `scenario_1: str | None = None` (Minimum Deliverables Scenario 1)
   - `scenario_2: str | None = None`
   - `scenario_3: str | None = None`
   - Validator: at least one scenario must be non-None

5. **`UsageRightsSet`** (BaseModel, frozen):
   - `paid_usage: UsageRightsDuration = UsageRightsDuration.not_required`
   - `whitelisting: UsageRightsDuration = UsageRightsDuration.not_required`
   - `organic_owned: UsageRightsDuration = UsageRightsDuration.not_required`

6. **`UsageRights`** (BaseModel, frozen):
   - `target: UsageRightsSet` (what we aim for)
   - `minimum: UsageRightsSet` (lowest acceptable)
   - Validator: for each right type, minimum duration must not exceed target duration (use enum ordering)

7. **`BudgetConstraints`** (BaseModel, frozen):
   - `campaign_budget: Decimal` (reject floats)
   - `target_influencer_count: int | None = None`
   - `target_cost_range: str | None = None` (text field from ClickUp)
   - `min_cost_per_influencer: Decimal | None = None` (floor - reject floats)
   - `max_cost_without_approval: Decimal | None = None` (ceiling - reject floats)
   - `cpm_target: Decimal | None = None` (reject floats, $20-$50 range)
   - `cpm_leniency_pct: Decimal | None = None` (reject floats, 0-100)

8. **`ProductLeverage`** (BaseModel, frozen):
   - `product_available: bool = False`
   - `product_description: str | None = None`
   - `product_monetary_value: Decimal | None = None` (reject floats)

9. **`CampaignRequirements`** (BaseModel, frozen):
   - `exclusivity_required: bool = False`
   - `exclusivity_term: str | None = None` (15/30/60/90 Days/Other)
   - `exclusivity_description: str | None = None`
   - `content_posted_organically: bool = False`
   - `content_approval_required: bool = False`
   - `revision_rounds: int = 0` (0, 1, or 2)
   - `raw_footage_required: str | None = None` (Yes/No/Preferred)
   - `content_delivery_date: str | None = None` (ISO date range string)
   - `content_publish_date: str | None = None` (ISO date range string)

10. **`CampaignBackground`** (BaseModel, frozen):
    - `client_website: str | None = None`
    - `campaign_manager: str | None = None`
    - `payment_methods: list[str] = []` (PayPal, ACH)
    - `payment_terms: str | None = None` (Net 7/15/30)

11. **`DistributionInfo`** (BaseModel, frozen):
    - `platform_distribution: str | None = None`
    - `market_distribution: str | None = None`
    - `influencer_size_distribution: str | None = None`

**Expand the existing `Campaign` model** to add these new optional fields while keeping all existing fields for backward compatibility:
- `background: CampaignBackground | None = None`
- `goals: CampaignGoals | None = None`
- `deliverables: DeliverableScenarios | None = None`
- `usage_rights: UsageRights | None = None`
- `budget_constraints: BudgetConstraints | None = None`
- `product_leverage: ProductLeverage | None = None`
- `requirements: CampaignRequirements | None = None`
- `distribution: DistributionInfo | None = None`

Keep existing fields (`budget`, `target_deliverables`, `cpm_range`, `platform`, `timeline`, `influencers`) intact so existing code continues to work. The new sub-models provide structured access to the same and additional data.

Remove the `must_have_at_least_one_influencer` validator from Campaign since the expanded form may create campaigns before influencer assignment. Make `influencers` default to `[]`.
  </action>
  <verify>
`uv run python -c "from negotiation.campaign.models import Campaign, CampaignGoals, DeliverableScenarios, UsageRights, BudgetConstraints, ProductLeverage, CampaignRequirements, CampaignBackground, DistributionInfo, UsageRightsDuration, OptimizeFor; print('All imports OK')"`
  </verify>
  <done>All 11 new types import successfully, Campaign model accepts both old-style (8 fields) and new-style (42 fields) construction without breaking existing code</done>
</task>
