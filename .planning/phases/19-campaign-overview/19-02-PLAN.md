<task type="auto">
  <name>Task 1: Create TypeScript types and polling hook</name>
  <files>frontend/src/types/campaign.ts, frontend/src/hooks/usePolling.ts</files>
  <action>
1. Create `frontend/src/types/campaign.ts`:
   - `CampaignStatusCounts`: { active_negotiations: number; agreed: number; escalated: number; rejected: number; total_influencers: number }
   - `CampaignMetrics`: { avg_cpm_achieved: number | null; pct_closed: number; budget_utilization: number | null }
   - `CampaignSummary`: { campaign_id: string; client_name: string; platform: string; budget: number; status_counts: CampaignStatusCounts; metrics: CampaignMetrics }
   - `CampaignListResponse`: { campaigns: CampaignSummary[]; total: number }
   - Export all as named exports.

2. Create `frontend/src/hooks/usePolling.ts`:
   - Generic hook: `usePolling<T>(url: string, intervalMs: number = 30000): { data: T | null; loading: boolean; error: string | null }`
   - On mount: fetch immediately, then setInterval at intervalMs.
   - Use `useEffect` with cleanup that calls `clearInterval`.
   - Use `useState` for data, loading, error.
   - Fetch with `fetch(url)`, parse JSON, set data. On error, set error string, keep previous data.
   - Set `loading: true` only on initial fetch (not on subsequent polls) to avoid UI flicker.
   - Export as named export.
  </action>
  <verify>Run `cd frontend && npx tsc --noEmit` to confirm types compile with no errors.</verify>
  <done>TypeScript types match API response shape exactly. usePolling hook fetches on mount and polls at configurable interval with proper cleanup.</done>
</task>
