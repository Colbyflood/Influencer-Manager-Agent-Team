   - Add `PAUSE = "pause"`, `RESUME = "resume"`, `STOP = "stop"` to the `NegotiationEvent` enum.
   - Add transitions from every non-terminal, non-paused, non-stopped state to PAUSED via "pause" event:
     - `(INITIAL_OFFER, "pause") -> PAUSED`
     - `(AWAITING_REPLY, "pause") -> PAUSED`
     - `(COUNTER_RECEIVED, "pause") -> PAUSED`
     - `(COUNTER_SENT, "pause") -> PAUSED`
     - `(ESCALATED, "pause") -> PAUSED`
     - `(STALE, "pause") -> PAUSED`
   - Add transition `(PAUSED, "resume") -> PAUSED` (placeholder; actual restore handled by machine).
   - Add transitions from every non-terminal state to STOPPED via "stop" event:
     - `(INITIAL_OFFER, "stop") -> STOPPED`
     - `(AWAITING_REPLY, "stop") -> STOPPED`
     - `(COUNTER_RECEIVED, "stop") -> STOPPED`
     - `(COUNTER_SENT, "stop") -> STOPPED`
     - `(ESCALATED, "stop") -> STOPPED`
     - `(STALE, "stop") -> STOPPED`
     - `(PAUSED, "stop") -> STOPPED`
   - Add `NegotiationState.STOPPED` to `TERMINAL_STATES` (STOPPED is terminal -- no further transitions).
   - Do NOT add PAUSED to TERMINAL_STATES (it should be resumable).

3. In `src/negotiation/state_machine/machine.py`:
   - Add a `_pre_pause_state` attribute (type `NegotiationState | None`, default `None`) to `__init__` and `from_snapshot`.
   - Update `from_snapshot` to accept an optional `pre_pause_state` parameter.
   - Add a `pre_pause_state` property that returns `_pre_pause_state`.
   - Add a `pause()` method that:
     - Checks current state is not terminal and not already PAUSED
     - Stores `self._state` into `self._pre_pause_state`
     - Calls `self.trigger("pause")` to record the transition
   - Add a `resume()` method that:
     - Checks current state IS PAUSED and `_pre_pause_state` is not None
     - Stores the pre_pause_state locally, sets `_pre_pause_state = None`
     - Records a history entry `(PAUSED, "resume", restored_state)`
     - Sets `self._state = restored_state` (bypasses normal trigger since PAUSED->resume isn't a real transition to a specific state)
   - Remove the `(PAUSED, "resume") -> PAUSED` placeholder from transitions.py if it was added -- instead, handle resume entirely in the machine's `resume()` method by directly manipulating `_state` and appending to `_history`. This avoids needing a fixed destination state in the transition map.
   - Add a `stop()` method that calls `self.trigger("stop")`.

  <action>
1. In `src/negotiation/api/negotiations.py`, add the following endpoints and models:

   Response models:
   - `ControlResponse(BaseModel)`: `thread_id: str`, `action: str`, `previous_state: str`, `new_state: str`
   - `BulkStopResponse(BaseModel)`: `agency_name: str`, `stopped_count: int`, `thread_ids: list[str]`
   - `BulkStopRequest(BaseModel)`: `agency_name: str`

   Endpoints:
   - `POST /campaigns/{campaign_id}/negotiations/{thread_id}/pause`:
     - Look up thread in `request.app.state.negotiation_states`
     - Verify thread exists and belongs to campaign_id (404 if not)
     - Verify state is not terminal and not already paused (409 Conflict if so, with descriptive message)
     - Call `state_machine.pause()`
     - Return `ControlResponse` with action="paused"

   - `POST /campaigns/{campaign_id}/negotiations/{thread_id}/resume`:
     - Look up thread, verify exists and belongs to campaign
     - Verify state IS paused (409 if not paused)
     - Call `state_machine.resume()`
     - Return `ControlResponse` with action="resumed"

   - `POST /campaigns/{campaign_id}/negotiations/{thread_id}/stop`:
     - Look up thread, verify exists and belongs to campaign
     - Verify state is not terminal (409 if already terminal)
     - Call `state_machine.stop()`
     - Return `ControlResponse` with action="stopped"

   - `POST /negotiations/stop-by-agency`:
     - Accept `BulkStopRequest` body with `agency_name` field
     - Iterate all entries in `request.app.state.negotiation_states`
     - For each where `context.get("agency_name") == agency_name` and state is not terminal:
       - Call `state_machine.stop()`
       - Collect thread_id
     - Return `BulkStopResponse` with count and list of stopped thread_ids
     - If no matching negotiations found, return 200 with stopped_count=0 (not an error)

2. In `src/negotiation/app.py`, in the `process_inbound_email` function, add a guard AFTER looking up thread_state (after line ~795 where `thread_state` is retrieved and confirmed non-None, and after `state_machine = thread_state["state_machine"]` on ~803):

   Add right after extracting state_machine:
   ```python
   # Guard: skip processing for paused or stopped negotiations
   if state_machine.state in (NegotiationState.PAUSED, NegotiationState.STOPPED):
       logger.info(
           "Negotiation is %s, skipping inbound email",
           state_machine.state.value,
           thread_id=inbound.thread_id,
       )
       return
   ```

   Import `NegotiationState` at the top of app.py if not already imported:
   `from negotiation.domain.types import NegotiationState`
