Implement `select_lever(ctx: NegotiationLeverContext) -> LeverResult` using RED-GREEN-REFACTOR.

The lever selection follows this priority order (from Phase 14 KB decision [14-01]):
deliverable tiers > usage rights > product > CPM sharing > syndication > exit

**Selection logic (deterministic, no LLM):**

1. **Cost floor check (NEG-12):** If `budget_constraints.min_cost_per_influencer` is set and `our_current_rate` would go below it, return `enforce_floor` with rate clamped to min_cost. The `lever_instructions` should say: "We are at our absolute minimum for this campaign. Frame this as the best we can offer given the campaign parameters."

2. **Cost ceiling check (NEG-12):** If `budget_constraints.max_cost_without_approval` is set and `their_rate` exceeds it, return `escalate_ceiling` with `should_escalate=True`. The `lever_instructions` should say: "This rate exceeds our approval threshold. Escalate to campaign manager for approval."

3. **Trade deliverables (NEG-09):** If `current_scenario < 3` and `deliverable_scenarios` has a next tier (scenario_2 or scenario_3), return `trade_deliverables` with the next scenario's text as `deliverables_summary` and recalculate rate at current CPM. The `lever_instructions` should describe: "We're adjusting the deliverable package to [scenario N text]. Frame this as streamlining the content scope while maintaining quality."

4. **Trade usage rights (NEG-10):** If `current_usage_tier == "target"` and `usage_rights` has a minimum that differs from target, return `trade_usage_rights`. The `lever_instructions` should say: "We're reducing usage rights duration to [minimum durations]. Position this as a cost-saving adjustment while keeping the content partnership intact."

5. **Offer product (NEG-11):** If `product_leverage` is available and `product_offered is False`, return `offer_product`. The `lever_instructions` should include: "In addition to the rate, mention that we can provide [product_description] valued at $[product_monetary_value]. Frame this as added value, not a rate replacement."

6. **Propose syndication (NEG-14):** If `deliverable_scenarios.content_syndication is True` and `syndication_proposed is False`, return `propose_syndication`. The `lever_instructions` should say: "Propose cross-posting content (e.g., IG to TikTok) rather than unique content per platform. Frame this as maximizing content reach with less production effort."

7. **Share CPM target (NEG-13):** If `budget_constraints.cpm_target` is set and `cpm_shared is False`, return `share_cpm_target`. The `lever_instructions` should say: "Share that our CPM target is $[cpm_target] to justify the budget constraint. Use this selectively -- only with motivated influencers who seem engaged but stuck on rate."

8. **Graceful exit (NEG-15):** If all applicable levers are exhausted, return `graceful_exit` with `should_exit=True`. The `lever_instructions` should say: "The deal economics don't work for this campaign. Send a warm, relationship-preserving exit. Express genuine appreciation, leave the door open for future campaigns, and wish them well."

**Opening high (NEG-08):** This is handled by the caller (start_negotiations_for_campaign in app.py) which should use scenario_1 deliverables and CPM floor rate. The engine itself handles subsequent rounds. Add a helper function `build_opening_context(campaign: Campaign, average_views: int) -> tuple[Decimal, str]` that returns (opening_rate_at_cpm_floor, scenario_1_deliverables_text). Use `calculate_rate(average_views, cpm_floor)` for the rate and `deliverable_scenarios.scenario_1` for deliverables. If no scenarios exist, fall back to `campaign.target_deliverables`.
